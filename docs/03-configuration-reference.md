# 03 - Configuration Reference

This document outlines the key configuration files and variables for the NetBird Infrastructure.

## Terraform Variables (`infrastructure/variables.tf`)

### Core Settings
| Variable | Description | Default | Required |
| :--- | :--- | :--- | :--- |
| `cloud_provider` | Target cloud (`aws`, `gcp`, `azure`) | - | Yes |
| `environment` | Environment name (e.g., `prod`, `dev`) | `prod` | No |
| `netbird_domain` | Main DNS domain (e.g., `vpn.example.com`) | - | Yes |
| `netbird_version` | NetBird version tag | `latest` | No |

### Database Configuration
| Variable | Description | Default | Options |
| :--- | :--- | :--- | :--- |
| `database_type` | Backend engine | `sqlite` | `sqlite`, `postgresql`, `mysql` |
| `database_mode` | Deployment mode | `existing` | `create`, `existing` |
| `enable_ha` | Enable High Availability features | `false` | `true`, `false` |
| `sqlite_database_path` | Path for SQLite DB file | `/var/lib/netbird/store.db` | - |

*(For external databases, see `variables.tf` for specific host/user/pass variables)*

### Keycloak Automation
| Variable | Description | Required |
| :--- | :--- | :--- |
| `keycloak_url` | URL of existing Keycloak (e.g., `https://auth.example.com`) | Yes |
| `keycloak_admin_client_id` | Admin CLI Client ID | Yes (default: `admin-cli`) |
| `keycloak_admin_client_secret`| Admin CLI Client Secret | Yes |
| `realm_name` | Name of Realm to create | No (default: `netbird`) |

## Ansible Configuration (`configuration/`)

### Inventory groups
The inventory is automatically generated by Terraform at `configuration/inventory/terraform_inventory.yaml`.

*   `management`: NetBird Management, Signal, and Dashboard services.
*   `reverse_proxy`: Caddy Load Balancers.
*   `relay`: NetBird Relay nodes.

### Role Variables (`group_vars/`)

#### `all.yml` (Global Defaults)
*   `database_engine`: Mirrors Terraform setting.
*   `coturn_password`: Shared secret for TURN.

#### `management.yml` (NetBird Backend)
*   `netbird_management_port`: 80
*   `netbird_signal_port`: 80
*   `netbird_dashboard_port`: 8080

#### `reverse_proxy.yml` (Caddy)
*   `caddy_version`: latest

## File Locations

| Service | File Type | Path | Description |
| :--- | :--- | :--- | :--- |
| **NetBird** | Config | `/etc/netbird/management.json` | Main generated config |
| **NetBird** | Docker | `/etc/netbird/docker-compose.yml` | Container orchestration |
| **NetBird** | Data | `/var/lib/netbird/` | Persistent data (SQLite, Certs) |
| **Caddy** | Config | `/etc/caddy/Caddyfile` | Reverse proxy rules |
| **UFW** | Config | `/etc/ufw/user.rules` | Persistent firewall rules |
