# Ansible Deployment Guide

## Overview

This directory contains Ansible automation for deploying NetBird to VMs discovered by Terraform.

## Structure

```
configuration/ansible/
├── ansible.cfg              # Ansible configuration
├── inventory/
│   ├── terraform_inventory.yaml  # Generated by Terraform
│   └── group_vars/
│       └── all.yml          # Global variables
├── playbooks/
│   └── site.yaml            # Main deployment playbook
└── roles/
    ├── common/              # System hardening
    ├── docker/              # Docker installation
    └── netbird/             # NetBird deployment
```

## Quick Start

### 1. Configure Variables

Edit `inventory/group_vars/all.yml`:

```yaml
netbird_domain: "netbird.example.com"
netbird_caddy_email: "admin@example.com"
netbird_database_backend: "sqlite"  # or "postgres"
```

### 2. Encrypt Secrets

```bash
ansible-vault encrypt inventory/group_vars/all.yml
```

### 3. Deploy

```bash
ansible-playbook playbooks/site.yaml --ask-vault-pass
```

## Database Configuration

### SQLite (Default)
No additional configuration required.

### PostgreSQL
```yaml
netbird_database_backend: "postgres"
netbird_postgres_password: "strong-password"
```

## Roles

- **common**: Firewall (UFW), SSH hardening, NTP (chrony)
- **docker**: Docker CE installation and configuration
- **netbird**: NetBird stack deployment with configurable database backend

## Tags

Run specific parts of the playbook:

```bash
# Only system hardening
ansible-playbook playbooks/site.yaml --tags common

# Only Docker installation
ansible-playbook playbooks/site.yaml --tags docker

# Only NetBird deployment
ansible-playbook playbooks/site.yaml --tags netbird
```

## Troubleshooting

### Vault Issues
```bash
# View encrypted file
ansible-vault view inventory/group_vars/all.yml

# Edit encrypted file
ansible-vault edit inventory/group_vars/all.yml
```

### Syntax Check
```bash
ansible-playbook --syntax-check playbooks/site.yaml
```

### Dry Run
```bash
ansible-playbook playbooks/site.yaml --check
```
