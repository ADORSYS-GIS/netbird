# Ansible Configuration

Ansible playbooks and roles for deploying NetBird infrastructure on VMs.

## Overview

This directory contains Ansible automation for deploying and managing NetBird services on existing VMs. The inventory is auto-generated by Terraform from `infrastructure/ansible-stack/`.

## Prerequisites

- Ansible 2.9+ installed locally
- SSH access to target VMs with sudo privileges
- Python 3.6+ on target VMs
- Inventory file generated by Terraform

## Quick Start

### 1. Generate Inventory

```bash
cd ../../infrastructure/ansible-stack
terraform apply
```

This creates `inventory/terraform_inventory.yaml` with all variables and hosts.

### 2. Deploy NetBird

```bash
ansible-playbook -i inventory/terraform_inventory.yaml playbooks/site.yml
```

### 3. Verify Deployment

```bash
ansible all -i inventory/terraform_inventory.yaml -m shell -a "docker ps"
```

## Playbooks

| Playbook | Description | Usage |
|----------|-------------|-------|
| `site.yml` | Full deployment | `ansible-playbook -i inventory/terraform_inventory.yaml playbooks/site.yml` |
| `upgrade.yml` | Rolling upgrade | `ansible-playbook -i inventory/terraform_inventory.yaml playbooks/upgrade.yml` |
| `cleanup.yml` | Remove all services | `ansible-playbook -i inventory/terraform_inventory.yaml playbooks/cleanup.yml` |
| `validate-security.yml` | Security validation | `ansible-playbook -i inventory/terraform_inventory.yaml playbooks/validate-security.yml` |
| `fix-deployment.yml` | Fix common issues | `ansible-playbook -i inventory/terraform_inventory.yaml playbooks/fix-deployment.yml` |

## Roles

| Role | Description |
|------|-------------|
| `common` | Base system setup (Docker, networking, packages) |
| `haproxy` | HAProxy load balancer with ACME.sh for TLS certificates |
| `netbird-mgmt` | NetBird management, signal, and dashboard services |
| `netbird-relay` | NetBird relay servers for peer connections |
| `pgbouncer` | PostgreSQL connection pooler |
| `reverse-proxy` | Caddy reverse proxy (alternative to HAProxy) |

## Deployment Tags

Use tags to deploy specific components:

| Deploy Tag | Cleanup Tag | Description |
|------------|-------------|-------------|
| `common` | `cleanup-common` | Base infrastructure |
| `haproxy` | `cleanup-haproxy` | HAProxy + ACME.sh + TLS |
| `management`, `mgmt` | `cleanup-management`, `cleanup-mgmt` | NetBird management services |
| `relay` | `cleanup-relay` | NetBird relay servers |
| `pgbouncer` | `cleanup-pgbouncer` | PostgreSQL connection pooler |
| `proxy` | `cleanup-proxy` | Any reverse proxy (HAProxy or Caddy) |

**Examples:**

```bash
# Deploy only HAProxy
ansible-playbook -i inventory/terraform_inventory.yaml playbooks/site.yml --tags haproxy

# Deploy only management services
ansible-playbook -i inventory/terraform_inventory.yaml playbooks/site.yml --tags management

# Cleanup HAProxy only
ansible-playbook -i inventory/terraform_inventory.yaml playbooks/cleanup.yml --tags cleanup-haproxy
```

## Configuration Variables

All configuration variables are set in `infrastructure/ansible-stack/terraform.tfvars` and automatically injected into the Ansible inventory.

### Key Variable Categories

**NetBird Core:**
- `netbird_domain` - Primary domain
- `netbird_version` - NetBird version
- `netbird_log_level` - Log level (info, debug, warn, error)
- `netbird_admin_email` - Admin email
- `netbird_admin_password` - Admin password

**Database:**
- `database_type` - postgresql or sqlite
- `database_dsn` - Full connection string
- `db_host`, `db_port`, `db_name`, `db_user`, `db_password` - Connection details
- `db_sslmode` - SSL mode (require, disable, verify-full)

**Keycloak (SSO):**
- `keycloak_url` - Keycloak server URL
- `keycloak_realm` - Realm name
- `keycloak_client_id` - Client ID
- `keycloak_backend_client_secret` - Backend client secret

**PgBouncer:**
- `enable_pgbouncer` - Enable connection pooling
- `pgbouncer_listen_port` - Listen port (default: 6432)
- `pgbouncer_pool_mode` - Pooling mode (transaction, session, statement)
- `pgbouncer_default_pool_size` - Default pool size per user/database
- `pgbouncer_max_client_conn` - Maximum client connections
- `pgbouncer_max_db_connections` - Maximum database connections

**HAProxy:**
- `proxy_type` - haproxy or caddy
- `haproxy_version` - HAProxy version
- `haproxy_stats_password` - Stats UI password

**ACME/TLS:**
- `acme_provider` - Certificate provider (letsencrypt)
- `acme_email` - Email for Let's Encrypt notifications

For complete variable reference, see [Infrastructure README](../../infrastructure/ansible-stack/README.md#configuration-options).

## Common Workflows

### Full Deployment

```bash
ansible-playbook -i inventory/terraform_inventory.yaml playbooks/site.yml
```

### Redeploy Specific Component

```bash
# Redeploy HAProxy
ansible-playbook -i inventory/terraform_inventory.yaml playbooks/cleanup.yml --tags cleanup-haproxy
ansible-playbook -i inventory/terraform_inventory.yaml playbooks/site.yml --tags haproxy

# Redeploy management services
ansible-playbook -i inventory/terraform_inventory.yaml playbooks/site.yml --tags management
```

### Update Configuration

```bash
# 1. Edit Terraform variables
vim ../../infrastructure/ansible-stack/terraform.tfvars

# 2. Regenerate inventory
cd ../../infrastructure/ansible-stack && terraform apply

# 3. Redeploy affected components
cd ../../configuration/ansible
ansible-playbook -i inventory/terraform_inventory.yaml playbooks/site.yml --tags management
```

### Enable PgBouncer

```bash
# 1. Set in terraform.tfvars
enable_pgbouncer = true

# 2. Apply and deploy
cd ../../infrastructure/ansible-stack && terraform apply
cd ../../configuration/ansible
ansible-playbook -i inventory/terraform_inventory.yaml playbooks/site.yml --tags pgbouncer
```

### Rolling Upgrade

```bash
# 1. Update version in terraform.tfvars
netbird_version = "0.32.0"

# 2. Regenerate inventory
cd ../../infrastructure/ansible-stack && terraform apply

# 3. Run upgrade playbook
cd ../../configuration/ansible
ansible-playbook -i inventory/terraform_inventory.yaml playbooks/upgrade.yml
```

## Troubleshooting

### Check Connectivity

```bash
ansible all -i inventory/terraform_inventory.yaml -m ping
```

### Verify Deployment

```bash
ansible all -i inventory/terraform_inventory.yaml -m shell -a "docker ps"
```

### View Logs

```bash
# Management logs
ansible management -i inventory/terraform_inventory.yaml -m shell -a "docker logs netbird-management"

# HAProxy logs
ansible reverse_proxy -i inventory/terraform_inventory.yaml -m shell -a "docker logs haproxy"

# PgBouncer logs
ansible management -i inventory/terraform_inventory.yaml -m shell -a "docker logs pgbouncer"
```

### Test Connectivity to Services

```bash
# Test management API
ansible management -i inventory/terraform_inventory.yaml -m shell -a "curl -sk https://localhost:443/api/status"

# Test database connection
ansible management -i inventory/terraform_inventory.yaml -m shell -a "docker exec netbird-management pg_isready -h \$DB_HOST -U \$DB_USER"
```

## Manual Inventory (Advanced)

If you need to create inventory manually without Terraform:

```bash
cp inventory/inventory.yaml.example inventory/my-inventory.yaml
# Edit inventory/my-inventory.yaml with your values
ansible-playbook -i inventory/my-inventory.yaml playbooks/site.yml
```
