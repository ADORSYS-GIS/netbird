# Ansible Configuration

Ansible playbooks and roles for deploying NetBird on existing VMs.

## Prerequisites

1. **Terraform inventory** generated by `infrastructure/ansible-stack/`
2. **SSH access** to all target VMs
3. **Ansible** installed locally (`pip install ansible`)

## Quick Start

### 1. Generate Inventory

First, run Terraform to discover VMs and generate inventory:

```bash
cd ../../infrastructure/ansible-stack
terraform apply
```

This creates `inventory/terraform_inventory.yaml` with discovered VMs.

### 2. Run Playbook

Deploy NetBird to all discovered VMs:

```bash
cd ../../configuration/ansible
ansible-playbook -i inventory/terraform_inventory.yaml playbooks/site.yml
```

### 3. Verify Deployment

```bash
ansible-playbook -i inventory/terraform_inventory.yaml playbooks/validate-security.yml
```

## Customization

### Adding Custom Roles

1. Create role directory: `mkdir -p roles/my-custom-role/tasks`
2. Add tasks: `vi roles/my-custom-role/tasks/main.yml`
3. Include in playbook:
   ```yaml
   - hosts: management
     roles:
       - my-custom-role
   ```

### Overriding Variables

Create `group_vars/custom.yml`:

```yaml
netbird_version: "0.28.0"
netbird_log_level: "debug"
```

### Customizing Firewall Rules

Edit `roles/ufw-netbird/defaults/main.yml`:

```yaml
netbird_allowed_ports:
  - 443
  - 80
  - 33073
  - 3478  # Add custom port
```

## Playbooks

### Main Deployment (`site.yml`)

Deploys complete NetBird infrastructure:
- Installs Docker
- Configures database clients
- Deploys NetBird services
- Sets up reverse proxy
- Configures firewalls

### Available Tags

Run specific parts of deployment:

```bash
# Only configure firewall
ansible-playbook -i inventory/terraform_inventory.yaml playbooks/site.yml --tags firewall

# Only deploy NetBird services (skip system setup)
ansible-playbook -i inventory/terraform_inventory.yaml playbooks/site.yml --skip-tags system
```

## Roles

### `common`
- System updates
- Essential packages
- User management

### `docker`
- Docker engine installation
- Docker Compose setup

### `netbird-server`
- NetBird Management service
- NetBird Signal service
- NetBird Dashboard

### `reverse-proxy`
- Caddy installation
- Automatic HTTPS (Let's Encrypt)
- Load balancing

### `ufw-netbird`
- UFW firewall rules
- Preserves existing rules
- NetBird-specific ports

### `database-client`
- PostgreSQL client
- MySQL client (if needed)
- Database connectivity verification

## Inventory Structure

Generated by Terraform (`inventory/terraform_inventory.yaml`):

```yaml
all:
  vars:
    netbird_domain: "vpn.example.com"
    database_dsn: "postgresql://..."
    keycloak_client_id: "..."
  
  children:
    management:
      hosts:
        netbird-mgmt-1:
          ansible_host: 10.0.1.10
    
    reverse_proxy:
      hosts:
        netbird-proxy-1:
          ansible_host: 203.0.113.10
```

## Troubleshooting

### Connection Issues

```bash
# Test connectivity
ansible all -i inventory/terraform_inventory.yaml -m ping

# Check SSH access
ansible all -i inventory/terraform_inventory.yaml -m command -a "hostname"
```

### Verbose Output

```bash
ansible-playbook -vvv -i inventory/terraform_inventory.yaml playbooks/site.yml
```

### Check Logs

```bash
# On target VM
sudo journalctl -u netbird-management -f
```
