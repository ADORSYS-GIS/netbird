# Ansible Configuration

Ansible playbooks and roles for deploying NetBird infrastructure on VMs.

## Prerequisites

- [Ansible](https://docs.ansible.com/ansible/latest/installation_guide/intro_installation.html) 2.9+ installed locally
- SSH access to target VMs with sudo privileges
- Python 3.6+ on target VMs
- Inventory file (auto-generated by Terraform or manually created from template)

## Roles

| Role | Description |
|------|-------------|
| `common` | Base system setup (Docker, networking, packages) |
| `haproxy` | HAProxy load balancer with ACME.sh for TLS certificates |
| `netbird-mgmt` | NetBird management, signal, and dashboard services |
| `netbird-relay` | NetBird relay servers for peer connections |
| `pgbouncer` | PostgreSQL connection pooler |
| `reverse-proxy` | Caddy reverse proxy (alternative to HAProxy) |

## Available Tags

| Deploy Tag | Cleanup Tag | Description |
|------------|-------------|-------------|
| `common` | `cleanup-common` | Base infrastructure |
| `haproxy` | `cleanup-haproxy` | HAProxy + ACME.sh + TLS |
| `management`, `mgmt` | `cleanup-management`, `cleanup-mgmt` | NetBird management services |
| `relay` | `cleanup-relay` | NetBird relay servers |
| `pgbouncer` | `cleanup-pgbouncer` | PostgreSQL connection pooler |
| `proxy` | `cleanup-proxy` | Any reverse proxy (HAProxy or Caddy) |

## Configuration

### Inventory Setup

#### Option 1: Terraform-Generated (Recommended)

```bash
cd infrastructure/ansible-stack
terraform apply
```

Creates `configuration/ansible/inventory/terraform_inventory.yaml` with all variables and hosts.

#### Option 2: Manual Inventory

```bash
cp inventory/inventory.yaml.example inventory/my-inventory.yaml
# Edit inventory/my-inventory.yaml with your values
```

### Inventory Variables

Set in `infrastructure/ansible-stack/terraform.tfvars` and injected into inventory:

#### NetBird Core

| Variable | Description | Example |
|----------|-------------|---------|
| `netbird_domain` | Primary domain | `vpn.example.com` |
| `netbird_version` | NetBird version | `0.31.0` |
| `netbird_log_level` | Log level | `info`, `debug`, `warn`, `error` |
| `netbird_admin_email` | Admin email | `admin@example.com` |
| `netbird_admin_password` | Admin password | `changeme` |

#### Database

| Variable | Description | Example |
|----------|-------------|---------|
| `database_type` | Database type | `postgresql`, `sqlite` |
| `database_dsn` | Full connection string | `postgresql://user:pass@host:5432/db` |
| `db_host` | Database host | `db.example.com` |
| `db_port` | Database port | `5432` |
| `db_name` | Database name | `netbird` |
| `db_user` | Database user | `netbird` |
| `db_password` | Database password | `password` |
| `db_sslmode` | SSL mode | `require`, `disable`, `verify-full` |

#### Keycloak (SSO)

| Variable | Description | Example |
|----------|-------------|---------|
| `keycloak_url` | Keycloak server URL | `https://auth.example.com` |
| `keycloak_realm` | Realm name | `netbird` |
| `keycloak_client_id` | Client ID | `netbird-client` |
| `keycloak_backend_client_secret` | Backend client secret | `secret` |

#### PgBouncer

| Variable | Description | Default |

| `pgbouncer_default_pool_size` | Default pool size per user/database | `25` |
| `pgbouncer_max_client_conn` | Maximum client connections | `1000` |
| `pgbouncer_max_db_connections` | Maximum database connections | `100` |

#### HAProxy

## Deployment

### Full Deployment

```bash
ansible-playbook playbooks/site.yml -i inventory/terraform_inventory.yaml
```

### Component Deployment

| Command | Description |
|---------|-------------|
| `ansible-playbook playbooks/site.yml -i inventory/terraform_inventory.yaml --tags haproxy` | Deploy HAProxy only |
| `ansible-playbook playbooks/site.yml -i inventory/terraform_inventory.yaml --tags management` | Deploy management services only |
| `ansible-playbook playbooks/site.yml -i inventory/terraform_inventory.yaml --tags relay` | Deploy relay servers only |
| `ansible-playbook playbooks/site.yml -i inventory/terraform_inventory.yaml --tags pgbouncer` | Deploy PgBouncer only |

### Cleanup

| Command | Description |
|---------|-------------|
| `ansible-playbook playbooks/cleanup.yml -i inventory/terraform_inventory.yaml` | Cleanup everything |
| `ansible-playbook playbooks/cleanup.yml -i inventory/terraform_inventory.yaml --tags cleanup-haproxy` | Cleanup HAProxy only |
| `ansible-playbook playbooks/cleanup.yml -i inventory/terraform_inventory.yaml --tags cleanup-management` | Cleanup management only |

### Upgrade

```bash
ansible-playbook playbooks/upgrade.yml -i inventory/terraform_inventory.yaml
```

## Common Workflows

### Redeploy HAProxy

```bash
ansible-playbook playbooks/cleanup.yml -i inventory/terraform_inventory.yaml --tags cleanup-haproxy
ansible-playbook playbooks/site.yml -i inventory/terraform_inventory.yaml --tags haproxy
```

### Update Configuration

```bash
# Edit Terraform variables
vim infrastructure/ansible-stack/terraform.tfvars

# Regenerate inventory
cd infrastructure/ansible-stack && terraform apply

# Redeploy affected components
cd ../../configuration/ansible
ansible-playbook playbooks/site.yml -i inventory/terraform_inventory.yaml --tags management
```

### Enable PgBouncer

```bash
# Set in terraform.tfvars
enable_pgbouncer = true

# Apply and deploy
cd infrastructure/ansible-stack && terraform apply
cd ../../configuration/ansible
ansible-playbook playbooks/site.yml -i inventory/terraform_inventory.yaml --tags pgbouncer
```

## Troubleshooting

| Command | Description |
|---------|-------------|
| `ansible all -i inventory/terraform_inventory.yaml -m ping` | Check connectivity |
| `ansible all -i inventory/terraform_inventory.yaml -m shell -a "docker ps"` | Verify deployment |
| `ansible management -i inventory/terraform_inventory.yaml -m shell -a "docker logs netbird-management"` | View logs |
